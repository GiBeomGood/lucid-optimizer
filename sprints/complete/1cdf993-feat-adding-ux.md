## 변경 동기

아이템 추가 중 잘못 입력한 값을 수정하려면 처음부터 다시 시작해야 했음. 또한 두 옵션을 다 채운 뒤 `Enter` 한 번에 바로 추가되다 보니 실수로 추가하는 경우가 있었음.

## 수정 내용

- **값 수정 중 Esc를 누르면 입력 전 상태로 복원됨** — 편집 중이던 내용이 버려지고, 기존에 저장된 값이 유지됨
- **옵션 종류(o)와 값(v)을 각각 따로 수정할 수 있음** — 이미 입력된 행에 커서를 두고 `o`로 옵션 종류, `v`로 값을 개별 편집 가능
- **`w` 키로 아이템 추가를 최종 확정** — 두 옵션을 모두 채운 상태에서 `w`를 눌러야 추가됨. 실수로 추가되는 경우 방지
- **옵션 하나를 완료하면 나머지 옵션으로 자동 이동** — 값 입력 후 Enter 시 아직 비어 있는 쪽으로 커서가 자동으로 넘어감

## 변경 파일

| 파일 | 변경 내용 |
|------|-----------|
| `src/app.rs` | 값 편집용 임시 버퍼(`val_draft`) 추가; 옵션 수정·확정 액션 분리 |
| `src/event.rs` | 포커스 상태별 키 맵핑 분리; `o`, `v`, `w` 키 추가 |
| `src/ui.rs` | 편집 중일 때 임시 버퍼 렌더링; 힌트 텍스트 갱신 |
