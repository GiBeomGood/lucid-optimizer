## 변경 동기

보유한 몽환의 결정이 여러 개일 때, 그 중 n개를 골라 쓰는 최적 조합을 수작업으로 비교하는 것은 사실상 불가능합니다.
조합 수가 수백~수천 개에 달하는 상황에서도 프로그램이 자동으로 전투력 기준 상위 k개 조합을 찾아 제시할 수 있도록
기능을 새롭게 추가했습니다.

## 추가 내용

### 핵심 로직 (`src/optimizer.rs` 신규)

- **전투력 계산** — 마력, 마력%, 크리티컬 데미지, 재사용 대기시간 감소, 숙련도를 합산해 전투력 수치를 산출합니다. 재사용 대기시간 감소는 최대 9초, 숙련도는 최대 100까지만 반영됩니다.
- **조합 생성** — m개 결정 중 n개를 뽑는 모든 C(m, n) 조합을 열거합니다.
- **크리티컬 확률 필터** — 기본 크리티컬 확률과 결정에 의한 크리티컬 확률의 합이 100% 미만인 조합은 제외합니다.
- **top-k 추출** — 유효 조합을 전투력 내림차순으로 정렬하고 상위 k개를 반환합니다. k번째와 동일한 전투력을 가진 조합도 모두 포함합니다.
- **백그라운드 스레드** — 계산이 무거울 수 있으므로 별도 스레드에서 실행하며, `mpsc` 채널로 진행 상황을 실시간 전달합니다.

### TUI 흐름 추가 (`src/app.rs`, `src/event.rs`, `src/ui.rs`)

| 화면 | 설명 |
|------|------|
| **홈 메뉴** | "최적 조합 찾기" 항목 추가 (세 번째 항목) |
| **입력 팝업** | 사용할 결정 수(n)와 top-k 값을 순서대로 입력 |
| **진행 화면** | 단계별 상태 메시지(한 줄, 교체 방식) + 전투력 계산 중 프로그레스 바 표시 |
| **결과 목록** | 조합을 전투력 내림차순으로 표시, 최하위 대비 배율(e.g. `112.3%`) 함께 표시 |
| **조합 상세** | 선택한 조합의 결정 목록을 읽기 전용으로 표시, 옵션1→옵션2 이름 기준 정렬 |
