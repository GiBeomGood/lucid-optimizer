## 변경 동기

아이템 목록 보기 상태에서 옵션이 `<옵션>: <값>` 형태로 표시될 때, 옵션 이름 길이가 제각각이어서 값의 시작 위치가 아이템마다 달랐음. 가독성이 낮고 시각적으로 어수선한 문제가 있었음.

## 수정 내용

- `build_option_spans`의 라벨 생성 방식을 변경
  - 기존: `format!("{}: ", opt.kind.display_name())` — 패딩 없음, 콜론 구분
  - 변경: `format!("{}  ", pad_kind(opt.kind.display_name()))` — 20컬럼 패딩, 공백 구분
- 이미 아이템 추가 모드에서 사용 중인 `pad_kind` 함수를 목록 뷰에도 적용하여, 두 화면에서 옵션-값 간격이 일관되게 표시됨
